Data types

Type name Mutability Representation Notes
Int Immutable

int

The division operator "/" always returns a float.

	>>> foo= 2
	>>> bar= 1
	>>> foo/bar
	>>> 2.0
Float Immutable

float
Bool Immutable

bool

 True / False
Tuple Immutable

tuple

 
	>>> foo= ()
	or
	>>> foo= tuple()

Since it's immutable, it can be used as keys for a dictionary in replace of list.
String Immutable

str

 
	>>> foo= ''
	>>> foo+ 1
	>>> TypeError: must be str, not int
	>>>
	>>> bar= foo+ '1'
	>>> bar
	>>> '1'
	>>> foo
	>>> ''
	>>> bar* 3
	>>> '111'
List Mutable

list

 
	>>> foo= []
	or
	>>> foo= list()
	>>>
	>>> foo.append('a')
	>>> foo
	>>> ['a']
Dictionary Mutable

dict

 
	>>> foo= dict()
	>>> foo
	>>> {}
	>>> foo['bar']= 1
	>>> foo
	>>> {'bar': 1}

Keys of dictionaries can only be immutable data types.
Set Mutable

set

 
	>>> foo= set()
	>>> foo
	>>> set()
	>>> foo.add('a')
	>>> foo
	>>> {'a'}

Set has no orders (but list does) and therefore indexing a set is inapplicable. (i.e., foo[0] is wrong)

Set contains no duplicates. E.g., 

	>>> foo= set()
	>>> foo.add('a')
	>>> foo.add('a')
	>>> foo
	>>> {'a'}
Names in Python are case-sensitive. E.g., list != List

  • root_window.attributes(arg1, arg2) reference page


    arg1: str, keywords prefixed with a hyphen.

    arg2: float. If it indicates a boolean value, 1.0 (or 1) stands for True and 0.0 (or 0) stands for False.

    • Making root_window transparent(in windows)

      
	root_window.attributes('-alpha', #alpha value)

      #alpha value: 0.0~ 1.0. 0.0 means fully transparent, whereas 1.0 means fully opaque

      E.g. 

      root_window.attributes('-alpha', 0.5)

    • Making root_window top most

      
	root_window.attributes('-topmost', 1)

  • root_window.geometry(arg1) reference page

    Determine the position of root_window.


    arg1: str, with the general form of 'wxh±x±y'

    • w: width, h: height, x: the character that separates them

    • +x: x pixels from the left side of the window to the left side of the desktop

    • -x: x pixels from the right side of the window to the right side of the desktop

    • +y: y pixels from the top of the window to the top of the desktop

    • -y: y pixels from the bottom of the window to the bottom of the desktop


    E.g. 

    root_window.geometry('120x50-0+20')

    120 and 50: window width and height, separated by x

    -0: there's no distance between the right side of the window and the right side of the desktop

    +20: 20 pixels from the top of the window to the top of the desktop


  • root_window.resizable(0, 0)

    Fix the size of the root window.
